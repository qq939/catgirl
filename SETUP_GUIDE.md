# 人猫替换网站 - 配置指南

## 核心改动：CLI 模式 → Workflow API 模式

之前的服务器依赖 `coze` CLI 命令行工具来生成图片，但 CLI 只在开发沙箱中可用，部署环境无法使用。

现在改用 **Coze Workflow API**，纯 HTTP 请求，任何部署环境都能用。

## 前置条件

你需要有一个 Coze 工作流（Workflow），它包含"图像生成"节点并能接收 prompt 和可选的参考图片。

## 第一步：创建 Coze 工作流

1. 打开 [coze.cn](https://www.coze.cn)，进入你的空间
2. 点击 **资源库** → **+ 新建资源** → **工作流**
3. 输入名称，如"人猫替换图片生成"

### 配置工作流节点

#### 开始节点（Start）
添加两个输入变量：
| 变量名 | 类型 | 说明 |
|--------|------|------|
| `prompt` | 文本（String） | 图片生成的提示词（必填） |
| `image_url` | 文本（String） | 参考图片的 URL（可选，图生图时使用） |

#### 图像生成节点（Image Generation）
1. 点击 **+** → 添加"图像生成"节点
2. 配置参数：
   - **正向提示词**：设置为 `{{prompt}}`（引用开始节点的 prompt 变量）
   - **参考图**：如果需要图生图，设置为 `{{image_url}}`（引用开始节点的 image_url 变量）
   - **图片比例**：3:4（竖构图）
3. 模型建议选择 Seedream 4.5 或更高版本

#### 结束节点（End）
- 输出变量名：`data`
- 值：选择图像生成节点的输出（通常是 `image` 或 `output`）

### 发布工作流
1. 点击 **试运行**，输入测试 prompt（如"一只可爱的猫咪"），确认能正常生成图片
2. 点击 **发布**

### 获取 Workflow ID
发布后，在浏览器地址栏中找到 workflow_id，格式类似：
```
https://www.coze.cn/space/xxx/project-ide/xxx/workflow/7461817035346149427
                                                                    ^^^^^^^^^^^^^^^^
                                                                    这就是 workflow_id
```

## 第二步：获取 PAT 令牌

1. 打开 [Coze PAT 管理](https://www.coze.cn/open/oauth/pats)
2. 点击 **添加新令牌**
3. 名称随意，有效期选最长，权限勾选 **工作流** 相关权限
4. 复制生成的 `pat_xxx` 令牌

## 第三步：配置环境变量

创建 `.env` 文件（参考 `.env.example`）：

```env
COZE_API_KEY=pat_你的令牌
COZE_WORKFLOW_ID=你的工作流ID
PORT=8082
```

## 第四步：启动服务

```bash
npm install
npm start
```

访问 http://localhost:8082 即可使用。

## 工作流输出格式适配

服务器会自动解析多种工作流输出格式：
- `{ "data": "https://..." }` - 直接 URL
- `{ "output": "https://..." }` - output 字段
- `{ "data": { "output": "https://..." } }` - 嵌套格式
- URL 数组格式

如果你的工作流输出格式不在以上范围内，请查看服务器日志中的 `[Workflow] 无法解析输出` 信息，然后修改 `cozeWorkflowRun` 函数中的解析逻辑。

## 图片上传流程

用户上传的图片处理流程：
1. 图片先上传到 OBS 中转服务，获取公开 URL
2. 公开 URL 作为 `image_url` 参数传给工作流
3. 工作流中的图像生成节点使用该 URL 作为参考图

## 常见问题

### Q: 工作流执行失败，提示权限不足
A: 确保 PAT 令牌有工作流执行权限，且令牌属于工作流所在空间。

### Q: 生成的图片质量不好
A: 在工作流的图像生成节点中调整模型版本和参数。Seedream 4.5 效果较好。

### Q: 图生图效果不理想，参考图没有被使用
A: 检查工作流中图像生成节点的参考图参数是否正确引用了 `{{image_url}}` 变量。确保提示词中包含对参考图的描述。
